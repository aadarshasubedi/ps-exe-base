ps-exe-base

Base library for Windows Form projects

View project on GitHub

Overview

This library is for Windows Forms projects. It is used to start your exe and handles arguments or loading of the startup form. It includes classes to manage external processes, setting fonts on forms and all controls and handling updates to UI threads from background worker processes.

Start Up

The start up routine assumes there are two ways of invoking your executable - normally (shows a UI) and through the command line with arguments (no UI). Your project needs to use the Sub Main Startup option. In Sub Main you should invoke the PS.ExeBase.Starter.Start method passing an instance of a class defined in your project which implements the IManager class.

Util

This class includes helper methods for common tasks in your project

Setting Fonts

Out of the box, Forms use the Microsoft Sans Serif font, which was fine for 1998, but is downright ugly nowadays. Use the InitializeFonts method in the Load event to set font on the form and all controls on the form. This uses the SystemFonts.MessageBoxFont font, which uses the base font for the OS on which the executable is running.

Message Boxes

I grew tired of typing the same code every time I wanted to show a message box. These helper methods save time by including the proper icons, buttons, etc.

Output Writer

This is a collection of classes to redirect output to a place of your choosing. This is handy when you have a background worker thread and you want to send output to the UI. It is a violation to access the UI thread from a background thread. Therefore, something as simple as updating a textbox from a background thread requires delegates, etc. This complexity is abstracted away through these classes.

ProcessRunner

This class makes it easy to initiate external processes. It wraps the System.Diagnostics.Process class. The StandardOutput and StandardError streams are redirected to single location for easy access. It also accepts an Output.IOutputWriter to present the output to the UI thread.
